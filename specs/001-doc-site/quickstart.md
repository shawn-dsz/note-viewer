# Quickstart Guide: Documentation Website

**Feature**: Documentation Website (001-doc-site)
**Branch**: `001-doc-site`
**Date**: 2026-01-14

## Overview

This guide will help you set up and start developing the documentation website. The site is a static site generator that converts markdown files in this repository into a browsable web interface.

---

## Prerequisites

Before starting, ensure you have:

- **Node.js** 18+ ([download](https://nodejs.org/))
- **pnpm** 8+ (package manager) - `npm install -g pnpm`
- **Git** - for version control

---

## Project Setup

### 1. Install Dependencies

From the repository root:

```bash
# Install build dependencies
pnpm install

# This will install:
# - vite (dev server + build tool)
# - typescript (type checking)
# - markdown-it (markdown parsing)
# - highlight.js (syntax highlighting)
# - flexsearch (client-side search)
# - tailwindcss (styling, optional)
# - vitest (testing)
# - playwright (e2e testing)
```

### 2. Project Structure

After setup, the project structure will be:

```text
.
├── specs/
```

## Development Workflow

### 1. Initial Build

Generate the static site from markdown files:

```bash
# Build the site
pnpm run build

# Output: dist/ directory with HTML, CSS, JS, and search-index.json
```

### 2. Start Development Server

Run the local development server with hot reload:

```bash
# Start dev server (default: http://localhost:5173)
pnpm run dev

# The server will:
# 1. Build the site initially
# 2. Watch for markdown file changes
# 3. Rebuild on changes
# 4. Auto-refresh the browser
```

### 3. Development Cycle

1. **Edit markdown files** in the repository
2. **See changes** - the dev server auto-rebuilds and browser refreshes
3. **Test locally** - verify rendering, navigation, and search
4. **Commit changes** - when satisfied

## Configuration

### Site Configuration

Edit `src/config/site.ts`:

```typescript
export const siteConfig = {
  name: "Operations Course Docs",
  baseUrl: "/", // or "/docs" for subdirectory deployment
  description: "MBA Operations Management course materials",
};
```

### Build Configuration

Edit `vite.config.ts` for build options:

```typescript
export default defineConfig({
  build: {
    outDir: "dist",
    emptyOutDir: true,
    // Increase if you have very large markdown files
    chunkSizeWarningLimit: 1000,
  },
});
```

## Deployment

### Static Hosting

The generated `dist/` directory can be deployed to any static hosting service:

**GitHub Pages**:
```bash
# Build
pnpm run build

# Deploy (using gh-pages)
pnpm exec gh-pages -d dist
```

**Netlify**:
- Connect your repository
- Set build command: `pnpm run build`
- Set publish directory: `dist`

**Vercel**:
- Connect your repository
- Set build command: `pnpm run build`
- Set output directory: `dist`

## Troubleshooting

### Build Fails

**Issue**: Build fails with "Cannot find module"

**Solution**:
```bash
# Clean install
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

### Dev Server Won't Start

**Issue**: Port 5173 already in use

**Solution**:
```bash
# Use different port
pnpm run dev -- --port 3000
```

### Search Not Working

**Issue**: Search returns no results

**Solution**:
1. Check `dist/search-index.json` exists
2. Check browser console for errors loading the file
3. Verify `includeExtensions` config includes `.md`

### Styles Not Applied

**Issue**: Page looks unstyled

**Solution**:
1. Check `dist/assets/` contains CSS files
2. Verify HTML references CSS correctly
3. Clear browser cache and hard refresh
